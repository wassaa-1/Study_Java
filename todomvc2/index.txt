step01(todomvc)에서의 한계점
1. 메인 화면(App.java)에 해당하는 App.java에서 데이터를 파싱(parsing), 연산, 출력 처리를 모두 다 하고 있음.
-> GodClass
2. 객체가 아닌 기본 자료형을 그대로 활용 중이라 불편함

Step02
1. Todo 데이터를 하나로 모아서 관리해주는 별도의 클래스 Todo 생성
2. 생성한 클래스(Todo)를 활용하여 데이터를 관리
3. 할일 순서 번호로 하나의 Todo 데이터를 조회하는 기능을 구현

개선된 부분
1. 기본 자료형을 하나의 참조 자료형(Todo)으로 모아서 관리하기 때문에
raw data(기본형 데이터)를 직접 건드리지 않음
